# 性能边界说明

## 目标性能

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 首页首次加载 | < 2s | 包含Docsify + JSON数据 |
| 搜索响应 | < 500ms | 输入后显示结果 |
| Skill卡片渲染 | < 1s | 单页20个卡片内 |
| 复制命令 | < 100ms | 点击后立即反馈 |

## 数据边界

| 指标 | 当前设计 | 上限建议 |
|------|----------|----------|
| Skill数量 | 8个示例 | 建议不超过100个 |
| 每Skill标签数 | 2个 | 建议不超过5个 |
| 描述长度 | < 200字符 | 建议不超过200字符 |
| 标签总数 | 4个 | 建议不超过20个 |

## 性能影响因素

### 加载时间
- Docsify CDN加载：~200ms
- JSON数据fetch：~50ms（本地）/ ~200ms（远程）
- 渲染DOM：~100ms

### 搜索性能
- JS遍历数组：O(n)，n=Skill数量
- 关键词匹配：每个Skill检查4个字段
- 建议：100个Skill内，纯JS搜索足够

### 内存占用
- JSON数据：~10KB
- DOM节点：每个卡片~50个节点
- Docsify：~100KB

## 优化建议

1. **减少Skill数量**：超过100个时考虑分页或懒加载
2. **CDN加速**：使用国内CDN加速Docsify加载
3. **JSON压缩**：移除不必要的字段可减少传输量
4. **避免重复渲染**：搜索时只更新列表区域

---

## 监控建议

部署后监控以下指标：
- 页面加载时间（通过浏览器Performance API）
- 搜索响应时间（通过JS计时）
- 错误率（通过console.error捕获）
